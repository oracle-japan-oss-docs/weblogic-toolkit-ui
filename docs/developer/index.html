開発者ガイド

WebLogic Kubernetes Toolkit UIは、WebLogicユーザーがアプリケーションをKubernetes環境で実行するように移動できるように設計されたデスクトップ・アプリケーションです。

開発環境の設定

  1. https://nodejs.org/en/download/から最新のLTSバージョンのnode.jsをダウンロードしてインストールします。

  2. プロキシ・サーバーでインターネットに接続する場合は、次の5つの環境変数を設定してプロキシを正しく構成します:

    ELECTRON_GET_USE_PROXY=true
GLOBAL_AGENT_HTTPS_PROXY=<proxy-url>
WKTUI_DEV_PROXY=<proxy-url>
HTTPS_PROXY=<proxy-url>
NO_PROXY=<list-of-no-proxy-hosts>

  3. コマンドラインを開き、次のコマンドを実行して、npmのバージョンを最新に更新します:

    sudo npm install --global npm

  4. 次のコマンドを実行して、グローバルgit構成を設定します:

    git config --global user.name "<your real name>"
git config --global user.email "<your-oracle-email-address>"
git config --global core.ignoreCase false

    WindowsまたはmacOSで開発している場合、この最後のコマンドは重要です。 はい、macOSファイル・システムはデフォルトで大/小文字が区別されません!

  5. GitHub (https://github.com/oracle/weblogic-toolkit-ui)でgitリポジトリをクローニングします。

  6. weblogic-toolkit-uiディレクトリでコマンドラインを開き、npm installを実行して、プロジェクトに必要なJavaScript依存関係をダウンロードしてインストールします。

  7. weblogic-toolkit-ui/electronディレクトリでコマンドラインを開き、npm run install-toolsを実行します。

  8. weblogic-toolkit-ui/webuiディレクトリでコマンドラインを開き、npm startを実行します。

  9. 前のステップのサーバーが完全に実行された後、weblogic-toolkit-ui/electronディレクトリでコマンドラインを開き、npm startを実行します。

WindowsまたはmacOSインストーラの構築

  1. 開発環境を設定し、アプリケーションが正しく動作していることを確認します。
  2. weblogic-toolkit-ui/electronディレクトリでコマンドラインを開き、npm run buildを実行します。
  3. weblogic-toolkit-ui/distディレクトリで実行可能ファイルおよびインストーラを検索します。

Linuxインストーラの構築

  1. 開発環境を設定し、アプリケーションが正しく動作していることを確認します。
  2. weblogic-toolkit-ui/webuiディレクトリでコマンドラインを開き、npm run build:releaseを実行します。
  3. weblogic-toolkit-ui/scriptsディレクトリのコマンドラインから、./devBuildLinuxInstallers.shを実行します。
  4. weblogic-toolkit-ui/distディレクトリで実行可能ファイルおよびインストーラを検索します。
