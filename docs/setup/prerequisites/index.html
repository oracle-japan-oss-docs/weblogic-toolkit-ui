機械翻訳について

前提条件

WebLogic Kubernetes Toolkit UIには、WKT UIアプリケーション機能を完全に使用するためにローカルにインストールする必要があるいくつかの前提条件があります。

これには次のものがあります。

  • Java Development Kit (JDK) - WebLogic Deploy ToolingおよびWebLogic Image Toolの両方で必要です。 ローカルWebLogic Serverインストールと互換性のあるJDKバージョンをインストールします。
  • WebLogic Server (WLS) - WebLogic Deploy Toolingで必要です。 WebLogic Serverバージョン12.2.1.3.0以降をインストールしてください。最新のPatch Set Updates (PSU)およびその他の推奨パッチをインストールすることをお薦めします。 アプリケーションでは、WebLogic Serverを使用して、クライアントとして使用するWebLogic Server Scripting Tool (WLST)およびその他の関連アーティファクトにアクセスできます。
  • WebLogic Remote Consoleバージョン2.3.0以降。 モデル・ビューでWDTモデルを作成するためのユーザー・インタフェースを提供します。
  • Docker (またはPodman) - イメージをローカルに作成し、イメージ・レジストリと対話するために必要です。 WindowsまたはmacOSで実行している場合は、Rancher Desktop、Docker Desktop、またはPodman Desktopを試してください。
  • kubectl - Kubernetesクライアント。バージョンはKubernetesクラスタ・バージョンと一致している必要があります。 Kubernetesクラスタとの対話に必要です。 詳細は、UIのKubernetes Client Configurationページを参照してください。
  • クラウド・プロバイダ・コマンドライン・インタフェース(CLI)ツール - Kubernetesクラスタがクラウド・プロバイダの管理対象Kubernetesサービスの1つで実行されている場合は、クラウド・プロバイダCLIをダウンロードしてインストールする必要があります。 これは通常、Kubernetesクラスタに対して適切に認証されるようにkubectlを構成するために必要です。 クラスタを操作するためのkubectlの構成の詳細は、Kubernetes Client Configurationページを参照してください。
  • helm - Helmバージョン3.9以降。 WebLogic Kubernetes Operatorをインストールし、イングレス・コントローラをインストールするために必要です。
  • openssl - イングレス・ルートのX.509 TLS証明書を生成するために使用し、アプリケーションに対して生成を依頼する必要があります。

WKT UIアプリケーションは、Electronフレームワークを使用して構築されているため、Electronでサポートされているプラットフォームおよびバージョンのみをサポートしています。プラットフォーム・サポートのリストを参照してください。 たとえば、Fedora 24以降に対するElectron要件のために、バージョン8.0以上Oracle Linux、RedHat LinuxおよびCentOS Linux「のみ」をサポートしています。

これらのローカル・ソフトウェアの依存関係に加えて、コンテナ化されたWebLogic ServerドメインとそのアプリケーションをデプロイできるKubernetesクラスタが必要です。 モデルがまだない場合は、WKT UIアプリケーションを使用して手動で記述するか、既存のドメインからモデルを検出できます。 ほとんどの場合、ドメインからモデルを検出するために使用できるWebLogicドメインから開始します。

Linuxの前提条件

  • Oracle、RedHat、CentOSなどのRPMベースのシステムの場合:

    • OSネイティブ資格証明ストアに資格証明を格納するには、デスクトップ環境が必要です。 システムにグラフィカル・デスクトップ環境がない場合は、Oracle LinuxにGNOMEデスクトップをインストールするなど、インストールできます:

    https://support.oracle.com/knowledge/Oracle%20Linux%20and%20Virtualization/2717454_1.html

    • デスクトップ環境を使用せずにリモート・マシンでWKT UIを実行し、X転送を介してアクセスするには、X転送が正常に機能していることを確認する必要があります。 たとえば、 
    sudo dnf install xterm
logout
ssh -X ...
xterm

  • UbuntuやDebianなどのDebianベースのシステムの場合:

    • OSネイティブ資格証明ストアに資格証明を格納するには、デスクトップ環境が必要です。 システムにグラフィカル・デスクトップ環境がない場合は、Ubuntu 20xにGNOMEをインストールするなど、インストールできます: 
    sudo apt install gnome-session gdm3
sudo reboot

    • デスクトップ環境を使用せずにリモート・マシンでWKT UIを実行し、X転送を介してアクセスするには、X転送が正常に機能していることを確認する必要があります。 たとえば、 

      sudo apt install xterm
logout
ssh -X ...
xterm
